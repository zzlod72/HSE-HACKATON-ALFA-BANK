# Модель дохода клиента — инструкция запуска

Команда «Хи-Хи Ха-Ха». Метрика: WMAE.

## Окружение
Python 3.10–3.14. Установка зависимостей:
```
pip install -r requirements.txt
```

## Запуск
1. Положите рядом с ноутбуком: `train.csv`, `test.csv`, `feature_description.csv`, `sample_submission.csv`.
2. Откройте `solution_v3.ipynb` в VS Code (Jupyter).
3. В ячейке `0. КОНФИГУРАЦИЯ`: `FAST_MODE = False`, `PROFILE = 'STRONG'` (~10–15 ч) или `'FULL'` (~20–30 ч).
4. Run All. Результат: `submission.csv` (формат `sep=';'`, `decimal=','`).

Перед отправкой убедитесь, что в выводе есть строка **«Санитарная проверка: ПРОЙДЕНА»**.

## Примечания
- Смоук-тест пайплайна: `FAST_MODE = True` (~2–3 мин).
- Прерывание безопасно: обучения и результаты Optuna кэшируются в `artifacts_v3/`, повторный Run All доучивает только недостающее.
- Версии бустингов закреплены (`==`) — на них решение протестировано и воспроизводимо. `optuna` нужен для тюнинга (без него берутся дефолтные параметры).
- Обучение только на CPU, GPU не требуется.
